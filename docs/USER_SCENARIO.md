# 마이링크 (MyLink) - 사용자 시나리오 (User Scenario)

*   **관련 PRD 문서:** `docs/PRD.md` (v1.0.0)

## 1. 방문자: 마이링크 페이지 방문 과정

*   사용자(방문자)는 크리에이터의 다양한 활동 채널을 한곳에서 **확인하기 위해** 인스타그램, 유튜브 등의 바이오에 적힌 마이링크 주소(`mylink.com/tester` 등)를 **클릭(접속)한다.**
*   방문자는 접속한 페이지에 등록된 링크가 하나도 없을 경우 혼란을 **방지하기 위해** 화면 중앙에 렌더링된 "아직 등록된 링크가 없습니다."라는 안내 문구를 **확인한다.**
*   방문자는 자신이 이동하고자 하는 목적지(포트폴리오, 쇼핑몰 등)를 직관적으로 **파악하기 위해** 렌더링된 각 링크 버튼의 텍스트 이름과 좌측 파비콘 아이콘을 **눈으로 탐색한다.**
*   방문자는 특정 외부 사이트 콘텐츠로 **이동하기 위해** 원하는 링크 버튼을 **탭(또는 클릭)한다.** (클릭 즉시 대상 URL로 새 창 또는 현재 창에서 이동됨)

## 2. 소유자: 링크 추가/수정/삭제 과정

### 2.1 인증 단계
*   소유자는 본인의 프로필 링크들을 **편집하기 위해** 화면 하단 구석 등에 일반 방문자의 주의를 끌지 않도록 작게 숨겨진 '자물쇠(🔒)' 아이콘을 클릭하여 구글(Google) 소셜 로그인을 **수행한다.**
*   소유자는 본인에게 수정 권한이 부여되었음을 **인지하기 위해** 화면에 표시되는 '편집 모드 활성화 (각 링크 옆에 수정/삭제 아이콘 노출)' 상태를 **확인한다.**

### 2.2 링크 추가 과정
*   소유자는 방문자들에게 새로운 웹페이지 경로를 **제공하기 위해** 화면 하단에 있는 [+ 새 링크 추가] 버튼을 **클릭한다.**
*   소유자는 새로운 링크의 정보를 **등록하기 위해** 즉시 나타나는 폼(Form)에 '타이틀'과 'URL'을 **입력하고 [저장] 아이콘을 누른다.** 
    * *(이때 시스템은 구글 파비콘 API를 통해 대상 URL의 아이콘을 자동으로 가져와 렌더링하며, 파비콘을 찾지 못한 경우 대체용 '기본 지구본(🌐) 아이콘'을 자동 적용한다.)*
*   소유자는 데이터가 백엔드(Firebase Firestore)에 정상적으로 반영되었는지 **안심하기 위해** 화면 하단에 잠깐 발생했다 사라지는 "✅ 저장되었습니다" 토스트(Toast) 알림을 **확인한다.**

### 2.3 링크 수정 과정 (인라인 편집)
*   소유자는 기존에 등록된 링크의 텍스트나 주소를 **변경하기 위해** 해당 링크 버튼 우측의 [수정 아이콘(✏️)]을 **클릭한다.**
*   소유자는 다른 관리자 페이지로 이탈하지 않고 현재 화면에서 곧바로 내용을 **수정하기 위해**, 기존 버튼 영역이 즉시 입력창(Input 필드)으로 전환된 곳에서 텍스트를 고치고 [저장 아이콘]을 **클릭한다.**
*   소유자는 수정 사항이 성공적으로 처리되었는지 **다시 한번 확인하기 위해** "✅ 저장되었습니다" 토스트 알림을 **확인한다.**

### 2.4 링크 삭제 과정
*   소유자는 더 이상 노출을 원치 않는 예전 링크를 **제거하기 위해** 해당 링크 항목 우측의 [삭제 아이콘(🗑️)]을 **클릭한다.**
*   소유자는 실수로 중요한 링크를 지워버리는 불상사를 **방지하기 위해** 화면 중앙에 팝업되는 "정말 삭제하시겠습니까?" 경고창(Confirm)에서 **[확인]을 누른다.** (확인 즉시 데이터베이스 단에서 완전 삭제되며, 화면 상에서 즉각 사라진다.)
