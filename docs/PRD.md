# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

*   **문서 버전:** v1.0.0
*   **최종 수정일:** 2026-04-05

## 1. 프로젝트 개요

*   **프로젝트명:** 마이링크 (MyLink)
*   **목적:** 크리에이터(본인)의 다양한 플랫폼, 포트폴리오 등의 링크들을 모아서 연결해주는 기능 중심의 멀티 링크 프로필 웹 사이트 구축.
*   **대상 사용자:** 크리에이터 중심. (현재는 본인 전용으로 구축하되, 크리에이터 친화적이고 직관적인 UI/UX를 메인 컨셉으로 지향함)
*   **개발 형태:** 기존 `my-profile` (Next.js) 레포지토리를 수정 및 확장하여 진행

## 2. 핵심 기능 목록

복잡한 관리자용 페이지나 별도의 모달(팝업) 창을 띄우지 않고, 화면 안에서 직접 조작하는 심플한 구조를 목표로 합니다.

### 2.1 필수 기능 (Must-Have)
*   **사용자 인증 (Auth):** 
    *   파이어베이스(Firebase) 기반의 구글 소셜 로그인
    *   소유자(본인) 로그인 시에만 편집용 UI 컨트롤이 노출됨
*   **직관적인 링크 관리 (인라인 에디팅 및 구글 API 연동):** 
    *   **단일 뷰어-에디터 통합:** 별도의 `/` (방문자) 와 `/admin` 분리 없이 메인 프로필 화면 하나에서 모든 작업 처리
    *   **링크 추가 및 삭제:** 버튼 클릭 후 필드에 간편하게 URL과 제목 기입하여 즉시 추가
    *   **링크 수정 (인라인 편집):** 기존 링크를 수정 버튼 클릭 시, 페이지를 이동하거나 팝업창을 열지 않고 **해당 링크 버튼 위치 자체가 즉시 입력창(Input 필드)으로 전환되는 '인라인 에디팅(Inline Editing)' 방식** 지원
    *   **자동 파비콘(Favicon) 처리:** 추가된 대상 웹사이트의 도메인을 구별하여 **'구글 제공 파비콘 API (Google Favicon API)'**를 사용해 자동으로 좌측 아이콘으로 삽입
*   **데이터베이스 트랜잭션:** 파이어베이스(Firestore)를 사용하여 데이터를 저장하고 실시간으로 불러옴

### 2.2 제외된 기능 (Not-in-Scope)
시스템을 빠르고 가볍게 구축하기 위해 다음 기능들은 제외합니다.
*   링크 순서를 변경(드래그 앤 드롭 등)하는 기능
*   개별 링크에 대한 활성화/비활성화 (숨기기) 토글 기능

## 3. 구조 상세 및 데이터 스키마

### 3.1 URL 라우팅 및 인터페이스 규칙
*   **고유 URL 할당:** 접속 주소(Path)의 슬러그(Slug)는 사용자의 `displayName` 값을 동일하게 활용합니다. (예: `mylink.com/tester` 의 `tester`)
*   **화면 렌더링 (방문자):** 접속 시 심플하고 깔끔한 링크 버튼들만 배열되어 보입니다.
*   **화면 렌더링 (크리에이터 로그인):** 각 링크에 마우스를 올렸을 때(또는 우측에 항상) 인라인 편집/삭제 아이콘 버튼이 노출됩니다.

### 3.2 데이터베이스(Firestore) 스키마 구조
*   `users` 콜렉션 하위에 `links` 서브 콜렉션이 중첩되는 형태로 모델링합니다.

*   **`users` (Root Collection)**
    *   `{uid}` (Document) - _소셜 로그인 UID_
        *   `displayName`: URL 슬러그(Slug)로 사용되는 시스템 고유 아이디. 가입/로그인 시 구글 지메일(Gmail) 계정의 앞부분('@' 기호 이전)을 추출하여 자동 할당됨 **(필수 적용 필드)**
        *   `username`: 실제 프로필 화면 중간에 텍스트로 노출될 사용자의 표시 이름(이름/별칭)
        *   `email`: 사용자 이메일
        *   **`links` (Sub-collection)**
            *   `{linkId}` (Document) - _문서 고유 ID_
                *   `title`: 프론트 화면에 노출될 링크 텍스트
                *   `url`: 대상 웹 주소(URL)
                *   `createdAt`: 등록된 생성 일자 시간 (정렬 시 활용 타임스탬프)
